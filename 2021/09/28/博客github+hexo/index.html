npm config set prefix “D:\nodejs\node_global”
npm config set cache “D:\nodejs\node_cache”
设置缓存和 module位置

配置文件 .npmrc 在c:/asus 下

安装hexo

hexo 是网页框架
在github建仓库 仓库名字为 你的账号名字+github.io

接下来就是安装Hexo，首先在D盘建立一个文件夹 Blog，点开 Blog 文件夹，鼠标右键打开 Git Bush Here，（不要放c盘，可能会造成权限不够改不了文件）

输入npm命令安装Hexo：

npm install -g hexo-cli 安装hexo
如果npm hexo命令找不到将npm hexo的路径添加到环境变量
系统变量中新建一个变量名为“NODE_PATH”，值为“D:\nodejs\node_global\node_modules”
然后编辑用户变量里的Path，将相应npm的路径改为：D:\nodejs\node_global，如下图：
hexo的路径为D:\nodejs\node_global\node_modules\hexo-cli\bin

  • hexo init 命令初始化博客

  • hexo g 静态部署：

  • hexo s 开始网页服务

浏览器输入 http://localhost:4000 就可以打开新部署的网页：

将Hexo部署到GitHub

现在回到我们的 Blog 文件夹，用笔记本打开 _config.yml 文件

deploy:
type: git
repository: git@github.com:angrycrowtakeflig/angrycrowtakeflig.github.io.git
#你的仓库地址 用ssh地址 我用https一直提交不上仓库
branch: main # 好像github将仓库主支改成main了建议用main,用master会创建另一个master分支
注意空格

然后回到 Blog 文件夹中，打开 Git Bash，安装Git部署插件，输入命令：

npm install hexo-deployer-git –save

然后分别输入以下三条命令：

hexo clean #清除缓存文件 db.json 和已生成的静态文件 public
hexo g #生成网站静态文件到默认设置的 public 文件夹(hexo generate 的缩写)
hexo d #自动生成网站静态文件，并部署到设定的仓库(hexo deploy 的缩写)

完成以后，打开浏览器，输入 https://xxx.github.io 就可以打开你的网页了：xxx为你的账号名字
